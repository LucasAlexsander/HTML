Responsive Web Design

O Responsive Web Design é sobre o uso de HTML e CSS para redimensionar, ocultar, encolher ou ampliar automaticamente um site, para que tenha uma boa aparencia em todos os dispositivos (desktops, tablets, celulares):

Configurando a janela de visualização

Para usarmos essa ferramenta presicamos adicionar a tag <meta> em todas as páginas da web.

Exemplo:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Isso definirá a janela de visualização da sua página, que fornecerá ao navegador instruções sobre como controlar as dimensões e o dimensionamento da página.

Imagens responsivas

São imagens que adaptam perfeitamente a qualquer tamanho de navegador.

Usando propriedade de largura

Se a propriedade width for definida como 100%, a imagem será responsiva e aumentará ou diminuirá. Em alguns casos, será melhor usar a propriedade max-width, porque a imagem podem ser ampliada e perder a sua qualidade.

Usando max-width

Se a propriedade for definida como 100%, a imagem será reduzida se necessário, mas nunca aumentará de tamanho para ser maior que o tamanho original.

Mostrando imagens diferentes dependendo da largura do monitor

Para isso vamos usar o elemento <picture>, ele permite definirmos diferentes imagens para diferentes tamanhos de janela dos navegadores.

Tamanho de texto responsivo

O tamanho do texto pode ser definido com uma unidade "vw", que significa a "largura da jenala de visualização".

Dessa forma, o tamanho do texto seguirá o tamanho da janela do navegador.

Olá mundo

Exemplo:

< h1 style="font-size: 10vw;"> Olá mundo </h1>

OBS: A janela de visualização é o tamanho da janela do navegador. 1vw=1% da largura da janela de visualização. Se a janela de visualização tiver 50 cm de largura, 1vw é 0,5 cm.

Semântica

Elemento semânticos descrevem claramente seu significado para o navegador e o desenvolvedor.
Não semânticos: <div> e <span> - não diz nada sobre seu conteúdo. Semânticos: <form>, <table> e <article> - claramente define seu conteúdo.

Elemento section

O elemento section define a seção do documento.

De acordo com a documentação HTML e W3C: "Uma seção é um agrupamento temático de conteúdo, normalmente com título."

Exemplos de section:

  • Capítulos
  • Introdução
  • Novos itens
  • Informações de Contato

Uma página normalmente pode ser dividida em seções de Introdução, conteúdo e informações de contato.

Elemento article

Especifica conteúdo autocontido e independente.

Um artigo deve fazer sentido por si só e deve ser possível distribuí-lo independente do resto do site.

Exemplos de onde o <articl> pode ser usado:

  • Postagem de fóruns
  • Postagem de blogs
  • Comentários do usúario
  • Cartões de produto
  • Artigos de jornal

Elemento <header>

Ele serve de contêiner para conteúdo introdutório ou conjunto de links de navegação.

Geralmente contém:

  • Um ou mais elementos de título (<h1> - <h6>)
  • Logotipo ou ícone
  • Informações de autoria

OBS: você pode ter varios header dento da uma única página. Mas não pode estar contido em outros, como por exemplo, footer, address ou outro header.


Elemento <footer>

Define o rodapé para um documento ou seção, normalmente contém:

  • Informações de autoria
  • Informações sobre direitos autorais
  • Informações de Contato
  • Mada de site
  • Voltar aos links principais
  • Documento relacionado

Elemento <nav>

Defini um conjunto de links de navegação.

Nem todos os links devem esta no elemento nav. O nav destina-se apenas ao bloco principal de links de navegação.

Navegadores, como leitores de tela para usuários com deficiência, podem usar este elemento para determinar se deve omitir a renderização inicial deste conteúdo.


Elemento <aside>

Este elemento define algum conteúdo além do conteúdo em que é colocado (como um barra lateral).

O <aside> deve estar relacionado indiretamente ao conteúdo circundante.


Elementos <figure> e <figcaption>

A tag <figure> especifica conteúdo independente, como ilustrações, diagramas, fotos, listagens de códigos, etc.

<figcaption> define um legenda para o elemento <figure>. <figcaption> pode ser colocado como primeiro ou último filho do <figure>.

O <img> define a imagem/ilustração real.

Guia de estilos e convenções de codificações

Um código em HTML limpo, organizado, consistente torna mais fácil para outras pessoas ler e entender.

Sempre declare o tipo de documento

Sempre declare o tipo de documento na primeira linha.
O tipo de documento correto para HTML é:

<!DOCTYPE html>

Use nomes de elementos em minúsculas

Html permite misturar letras maiúculas com minúsculas em nomes de elementos. No entanto, recomenda-se o uso de nomes com letras minúsculas, por que:

  • Misturar nomes em maiúsculas e minúsculas parece ruim
  • Os desenvolvedores normalmente usam nomes em letras minúsculas
  • Letras minúsculas parecem mais limpas
  • Letras minúsculas são mais fáceis de escrever

Sempre feche todos os elementos

Sempre que você abrir um p feche-o, isso tem que ser usado em todos os elementos.


Sempre cite valores de atributos

HTML permite valores sem aspas.

No entando recomenda-se, citar valores de atributos, por que:

  • Os desenvolvedores normalmente citam valores de atibutos
  • Os valores citados são mais fáceis de ler
  • Você DEVE usar aspas se o valor contiver espaços

Sempre especifique alt, largura e altura para imagens

Ao usarmos o alt ajudará caso a imagem não carregue. E usar o width e height fará com que reduza a tremulação, porque o navegador pode reservar espaço para a imagem antes de carregar.


Espaços e sinais de igualdade

Usar menos espaço é mais fácil para ler e agrupar melhor as entidades.


Evite linhas de códigos longas

Não é conveniente rolar para a direita e para a esquerda para ler códigos HTML.


Linhas em branco e recuo

Não adicione linhas em branco, espaços ou recuo sem motivo.

Para facilitar a leitura, adicione linhas me branco para separar blocos de códigos grandes ou lógicos.

Para faciliatar a leitura, adicione dois espaços de recuo. Não Use a tecla tab.


Nunca pule o elemento <title>

O title é obrigatorio em HTML

O conteúdo do título é muito importante para a otimização do mecanismo de pesquisa (SEO)! O título da página é usado pelos algoritmos do mecanismo de pesquisa para decidir a ordem ao listar as páginas nos resultados da pesquisa

O title:

  • Define um título na barra de ferramentas do navegador
  • Fornece um título para a página quando ela é adicionada aos favoritos
  • Exibe um título para a página nos resultados do mecanismo de pesquisa

Omitir html e body

Sempre adicione as tag html e body.

A não omitir dessas tags podem causar o mal funcionamento dos navegadores.


Omitindo head?

Deve ser adicionada igual as tags html e body


Fechar os elementos HTML vazios

É opcional fechar elementos vazios.

Se você esperar que um software XML / XHTML acesse sua página, mantanha a barra de fechamento (/), porque ela é necessária em XML e XHTML.


Adicione o atributo lang

Você deve sempre incluir o lang dentro do html, para declarar o idioma da página Web. Isso serve para auxiliar os mecanismos de pesquisa e navegadores.

Extensões de arquivos

Os arquivos HTML devem ter uma extensão .html(.htm também é permitido). Não existe diferença entre .htm para .html, ambos serão tratados como HTML por qualquer navegador.

Os arquivos CSS devem ter uma extensão .css.

Os arquivos JavaScript devem ter uma extensão .js.


Nome de arquivos padrão

Quando a URL não especifica um nome de arquivo no final (https:google.com/), o servidor apenas adiciona um nome de arquivo padrão, como "index.htm", "index.html", "default.htm" ou "defautl.html".

Se o seu servidor estiver configurado apenas com "index.html" como nome de arquivo padrão, seu arquivo deve ser nomeado "index.html" e não "default.html"

No entando, os servidores podem ser configurados com mais de um nome de arquivo padrão; normalmente você pode configurar quantos nomes de arquivo padrão desejar.


Entidades HTML

Alguns caracteres são reservados para HTML. Por exemplo: se você usar os sinais de maior (<) ou o sinal de menos (>) em seu texto, o navegador pode acabar misturando eles com tags.

As entidades de caracteres são usados para exibir caracteres reservados em HTML.

Uma entidade HTML se parece com isso:


            &entity_name;
            OU 
            &#entidy_number;

Para escrevermos uma sinal de menor (>), demos escrever: & gt; ou & #60;.

Vantegem de usar um nome de entidade: O nome de uma entidade é fácil de lembrar.

Desvantagem de usar um nome de entidade: Os navegadores podem não oferecer suporte a todos os nomes de entidade, mas o suporte para número de entidade é sempre bom.

Espaços inquebrável

Uma entidade comumente usada no HTML é o espaço sem quebra: & nbsp;. Este espaço não separável é um espaço que não se quebra em uma nova linha.

Duas palavras separadas por um espaço não separável ficarão grudadas (não quebrarão em um nova linha). Isso é útil quando quebrar as palavras pode ser perturbador.

Exemplos:

  • §   10
  • 10 km  / 
  • 22h

Outro uso do espaço separável é evitar que o navegador truncem os espaços nas páginas HTML. Por exemplo, você coloca 10 espaços em um texto, o navegador irá retirar 9 deles. Para adicionar espaços reais ao seu texto, para isso você pode usar a entidade & nbsp;.

OBS: o hífen sem quebra (& # 8290;) é usado para definir um caracter de hífen (-) que não se divide em uma nova linhas.

Emojis no HTML

Emojis são personagens do conjunto de caracteres UTF-8

O que são Emojis

Podem parecer imagens ou icones, mas são letras (caracteres) do conjunto de caracteres UTF-8 (Unicode).

UTF-8 cobre quase todos os personagens e símbolos do mundo

O atributo do conjunto de caracteres HTML

Para exibir uma página corretamente, um navegador deve conhecer o conjunto de caracteres usado na página. Para fazermos ele reconhecer usamos a tag <meta>.

<meta charset="UTF-8">

Personagens UTF-8

Muitos caracteres UTF-8 não podem ser digitados no teclado, mas sempre podem ser selecionados usando números (chamados de números de entidade)

  • A é 65
  • B é 66
  • C é 67

Exemplo explicado

O <meta charset="UTF-8"> define o conjunto de caracteres.

Para permitir que o navegador entenda que você está exibindo um caractere, você deve iniciar o número da entidade com &# e terminar com ; (ponto-e-virgula).

Personagens - Emoji

  • 😄 é 128516
  • 😍 é 128525

Como emoji são caracteres eles podem ser estilizados, copiados como qualquer outro caractere em HTML.


Codificação HTML (conjuntos de caracteres)

Para exibir uma página corretamente, um navegador da web deve saber qual conjunto de caracteres usar.

De ASCII para UTF-8

ASCII foi o primeiro padrão de codificação de caracteres. O ASCII definiu 128 caracteres diferentes que podem ser usados na Internet: números (0-9), letras inglesas (AZ) e alguns caracteres especiais como ! $ + - ( ) @ < >.

ISO-8859-1 era o conjunto de caracteres padrão para HTML 4. Este conjunto de caracteres suportava 256 códigos de caracteres diferentes. HTML 4 também suporta UTF-8.

ANSI (Windows-1252) era o conjunto de caracteres originais do Windows. ANSI é idêntico ao ISO-8850-1, exceto que ANSI tem 32 caracteres extras.

A especificação HTML 5 incentiva os desenvolvedores da web a usar o conjunto de caracteres UTF-8, que cobre quase todos os caracteres e símbolos do mundo!


Codificação de URL

URL é uma outra página para um endereço Web.

Uma URL pode ser composta de palavras (por exemplo, google.com) ou um endereço de protocolo da Internet (IP) (Por exemplo, 192.68.20.50).

A maioria das pessoas insere o nome ao navegador, porque os nomes são mais fáceis de lembrar do que os números.

URL - Localizador Uniforme de Recursos

Os navegadores Web solicitam páginas de serviços da Web usando URL.

Um Uniform Resource Locator (URL) é usado para entereçar um documento (ou outros dados) na Web.

Um endereço da Web como https://google.com segue estas regras de sintaxe:

scheme://prefix.domain:port/path/filename

  • scheme - esquema: define o tipo de serviço da Internet (o mais comum é http ou https
  • prefix - prefixo: define um prefixo de domínio (o padrão para http é www
  • domain - domínio: define o nome de domínio da Internet (como google.com)
  • port - porta: define o número da porta no host (o padrão para http é 80)
  • path - caminho: define um caminho no servidor (se omitido: o diretório raiz do site)
  • filename - nome do arquivo: define o nome de um documento ou recurso

Esquema de URL comuns

A tabela abaixo lista alguns esquemas comuns:

Codificação da URL

Os URL só podem ser enviados pela Internet usando o conjunto de caracteres ASCII. Se um URL contém caracteres fora do conjunto ASCII, o URL deve ser convertido.

A codificação de URL converte caracteres não ASCII em um formato que pode ser transmitido pela Internet.

A codificação de URL substitui caracteres não ASCII por um "%" seguindo por dígitos hexadecimais.

Os URLs podem conter espaços. A codificaçao de URL pode normalmente substituir um espaço por um sinal de mais (+) ou % 20.


HTML vs XHTML

XHTML é uma versão de HTML mais restrita e mais baseada em XML.

O que é XHTML

  • XHTML está para E X tensible H Yper T ext M arkup L anguage
  • XHTML é uma versão de HTML mais restrita e baseada em XML
  • XHTML é HTML definido como um aplicativo XML
  • XHTML é compatível com todos os principais navegadores

Por que XHTML?

XML é uma linguagem de marcação em que todos os documentos devem ser marcados corretamente (ser "bem formatados").

O XHTML foi desenvolvido para tornar o HTML mais extensível e flexível para trabalhar com outros formatos de dados (como XML). Além disso, os navegadores ignoram os erros nas páginas HTML e tentam exibir o site mesmo se houver alguns erros de marcação. Portanto, o XHTML vem com um tratamento de erros muito mais rígido.

As diferenças mais importantes do HTML

  • <! DOCTYPE> é obrigatório
  • O atributo xmlns em <html> é obrigatório
  • <html>, <head>, <title> e <body> são obrigatório
  • Os elementos devem sempre ser alinhados de maneira adequada
  • Os elementos devem estar sempre fechados
  • Os elementos devem estar sempre em minúsculas
  • Os nomes dos atributos devem sermpre estar em minúsculas
  • Os valores dos atributos devem estar sempre entre aspas
  • A minimização de atributos é proibida

XHTML - <!DOCTYPE...> é obrigatório

Um documento XHTML deve ter uma declaração XHTML <!DOCTYPE>.

Os elementos <html>, <head>, <title> e <body> também devem estar presentes, e o atributo xmlns em <html> deve especificar p namespace xml para o documento.

Os elementos XHTML devem ser aninhados corretamente:

<b><i> Algum texto <i><b>

Os elementos devem ser sempre fechados

Os elementos vazios de XHTML devem ser sempre fechados

  • Uma quebra de linha: <br />
  • Uma linha horzontal: <hr />
  • Uma imagem: <img src="path" alt="nome" />

Os elementos sempre em letras minúsculas

Valores de atributos XHTML devem ser citados

Em XHTML, os valores dos atributos sempre devem estar entre aspas.

A minimização de atributos em XHTML é proibida
