Formulário HTML

Um formulário é usado para coletar a entrada do usuário. A entrada do usuário é geralemente enviada a um servidor para processamento.

O elemento <form>

Este elemento é usado para criar um formulário HTML para entrada do usuário. É um contêiner para diferentes tipos de elementos de entrada, como: campos de texto, caixas de seleção, botões de opção, botões de envio, etc.

O elemento <label>

A tag label define um rótulo para muitos elementos do formulário. Sendo muito útil para usuário de leitores de tela, porque o leito de tela lerá o rótulo em voz alta quando o usuário focar no elemento de entrada.

O label também ajuda os usuário que têm dificuldade em clicar em regiões muito pequenas (como botões de rádio ou caixas de seleção) - porque quando o usuário clica dentro do label, ele altera o botão de rádio / caixa de seleção.

O atributo for elemento do label deve ser igual o atributo id do elemento input para uni-los.

Este atributo como já vimos ele especifica onde exibirá a resposta recebida após o envio do formulário.

O atributo method

Este atributo especifica o método HTTP a ser usado ao enviar os dados do formulário.

Os formulários podem ser enviados como váriaveis de URL (com method="get") ou como transação HTTP post (com method="post"). Mas o método padrão para enviar dados do formulário é GET.

Tendo como uma diferença entre eles, após o envio a resposta exibida no GET aparece o valor na barra de endereço na nova página. Já o POST não, fica invisível.

Notas sobre o GET:

  • Acrescenta os dados do formulário ao URL, em pares nomes/valores
  • NUNCA use GET para enviar dados confidencias! (os dados do formulário enviado são visíveis no URL!)
  • O comprimento de um URL é limitado (2.048 caracteres)
  • Útil para envios de formulários onde um usuário deseja marcar o resultado
  • GET é bom para dados não seguros, como string de consulta no Google

Notas no POST

  • Acrescenta os dados do formulário dentro do corpo da solicitação HTTP (os dados do formulário enviado não são mostrados no URL)
  • O POST não tem limitações de tamanho e pode ser usado para enviar grandes quantidade de dados.
  • Os envios de formulários com POST não podem ser marcados

Dica: Sempre use o POST se os dados do formulário contiverem informções confidenciais ou pessoais!

Atributo de preenchimento automático

O autocomplete especifica se um formulário deve ter o preenchimento automático ativado ou desativado.

Quando o preenchimento automático está ativado, o navegador completa os valores automaticamente com base nos valore que o usuário inseriu anteriormente.

<form action="/action_page.php" autocomplete="on">

O atributo Novalidate

É um atributo booleano.

Quando presente, especifica que os dados do formulário (entrada) não devem ser validados ao serem enviados.

Formulário com atribulo novalidate

<form action="/action_page.php" novalidate">

Input

O valor mais usado, pode ser exibido de várias formas, dependendo do type.

Label

Define o rótulo de vários elementos do formulário. Sendo útil para leitores de tela, porque o leitor lerá o rótulo em voz alta quando o usuário focar no elemento de entrada.

Ajudando também os usuário com dificuldade em clicar em regiões muito pequenas (como botões de rádio ou caixas de seleção) - porque quando o usuário clica no texto dentro do label, ele alterna o botão de rádio / caixa de seleção.

O for da label deve ser igual ao id do input para uni-los.

Select

Define uma lista suspensa.


Os option definem uma opção que pode ser selecionada.

Por padrão, o primeiro item da lista suspensa é selecionado. Para definir uma opção pré-selecionada, adione o atributo à opção.

<option value="fox" selected> Fox </option>

Valores visíveis:

Use o size para especificar o número de valores visíveis:


<select name="carros" id="cars" size="3">

Permitir váras seleções:

Use o multiple para permitir que o usuário possa selecionar mais de uma valor


Para selecionar mais de UM use o CRTL

Textarea

Define um campo de entrada multilinhas (uma área de texto):



<textarea name="mensagem" rows="10" cols="30">
O cato está brincando no jardin.
</textarea>

O rows especifica o número visíveis de linhas em uma área de texto.

O cols especifica a largura visíveis de uma área de texto.

Também podemos definir o tamanho através do CSS, width e o height.

Button

Define um botão clicável.

<button type="button" onclick="alert('Opa, beleza?')">
Clique aqui!
</button>

Nota: Sempre especifique o type para elemento de botão. Navegadores diferentes podem usar tipos padrões diferentes para o elemento de botão.

Atributos Input

Veremos diferentes atributos para o Input


Value

Especifica um valor inicial para um campo de entrada. Um nome na barra de digitação.


Readonly

Especifica um campo para somente leitura, fazendo com que o usuário não possa modificar os dados (porém o usuário pode tabular até ele, destacá-lo e copiar o texto dele).


Disabled

Especifica um campo que deve ser desativado. Um campo de entrada desativado é inutilizável e não clicável.

Um valor de um campo de entrada desabilitado não será enviado ao enviar o formulário.


Size

Especifica a largura visível, em caracteres, de um campo de entrada. O valor padrão para size é 20.

OBS:o size funciona com os seguintes tipos de entrada: text, search, tel, url, email e password.


Maxlength

Especifica o número de caracteres permitidos em um campo de entrada.

OBS: quando o maxlength é definido, o campo de entrada não aceita mais do que o número especifico de caracteres. No entando, este atributo não fornece nenhum feedback. Portanto, se você desejar alertar o usuário, deve escrever o código JavaScript.


Max e Min

Especifica valores máximos e minímos para um campo de entrada. Estes atributos funcionam para os seguintes tipos de entrada: number, range, date, datetime-local, month, time e week


Multiple

Especifica que o usuário tem permissão para inserir mais de um valor em um campo de entrada. Funcionando com os seguintes tipos de entrada: email e file.



Pattern

Especifica uma expressão regular com qual o valor do campo de entrada é verificado, quando o formulário é enviado.

Funciona com os seguintes elementos: text, date, search, url, tel, email e password.

Dica: Use o atributo title global para descrever o padrão para ajudar o usuário.


Placeholder

Especifica uma pequena dica que descreve o valor esperado de um campo de entrada (um valor de amostra ou uma breve descrição do formato esperado).

A dica curta é exibida no campo de entrada antes que o usuário digite algo.

Funcionando com os seguintes elementos: text, search, tel, email, url e password.


Required

Especifica que o campo de entrada deve ser preenchido antes de enviar o formulário. Funcionando com os seguintes tipos de entrada: text, search, utl, tel, email, password, number, radio, file, checkbox e data pickers.


Step

Especifica os intervalos de números legais para um campo de entrada. Exemplo: se step="3", os números legais podem ser -3, 0, 3, 6,... .

Dica: Este atributo pode ser juntado com os atributos max e min para criar um intervalo de valores legais.

step funciona com os seguintes tipos de entrada: number, range, date, datetime-local, month, time and week.

Nota:As restrições de entradas não são infalíveis e o JavaScript fornece muitas maneiras de adicionar entras ilegais. Para restringir a entrada com segurança, ela também deve ser verificada pelo receptor (o servidor).


Autofocus

Especifica um campo de entrada que deve obter foco automaticamente quando a página é carregada.


Height e width

Especificam a altura e a largura de um <input type="image">.

Dica: Sempre especifique os atributos de altura e largura das imagens. Se a altura forem definidas, o espaço necessario será reservado quando a página for carregada. Sem esses atributos, o navegador não sabe o tamanho da imagem e não pode reservar o espaço apropriado para ela. O efeito será que o layout da página mudará durante o carregamento (enquanto as imagens carregam).


list

Se refere ao elemento datalist que contém predefinidas para um elemento <input>


Autocomplete

Especifica um campo de entrada que deve ter o preenchimento automático ativado ou desativado.

O preenchimento automático permite que o navagedor preveja o valor. Quando um usuário começa a digitar em um campo, o navegador deve exibir opções para preencher o campo, com base nos valores digitados anteriormente.

Funcionado com os segintes tipos: text, search, url, tel, email, password, range, cor e datepickers.

Dica:em alguns navegadores, pode ser necessário ativar uma função de preenchimento automático para que isso funcione.


Form Atributos

O atributo <form> especifica o formulário ao qual o elemento <input> pertence.

O valor deste atributo deve ser igual ao atributo id do elemento <form> ao qual pertence.



O campo "Sobrenome" está fora do elemento do formulário, mas ainda faz parte do formulário.


Formaction

Especifica a url do arquivo que irá precessar a entrada quando o formulário for enviado.

Nota: este atributo substitui o action do <form>

Formaction funciona com os seguintes tipos de entrada: enviar e imagem.







Formenctype

Especifica como os dados do formulário devem ser codificados quando enviados (somente para formulário com method="post").

Nota: este atributo substitui o atributo enctype do form.

O formenctype funciona com os seguintes tipos de entradas: enviar e imagem.


Formmethod

Define o método HTTP para enviar dados do formulário para URL de ação.

OBS: este atributo substitui o atributo de método do form.

O formmethod funciona com os seguintes tipos de entrada: enviar e iamgem.

Os dados do formulário pode ser enviados como váriavei de URL (método ="get") ou como uma transação HTTP post (método = "post").

Observação sobre o método "get":

  • Este método anexa os dados do formulário ao URL em pares nome / valor
  • Este método é útil para envios de formulários onde um usuários deseja marcar o resultado
  • Há um limite para a quantidade de dados que você pode colocar em uma URL (varia entre os navegadores)
  • Nunca use o método "get" para passar informações confidenciais! (a senha ou outras informções confidenciais estarão visíveis na barra de endereço do navegador)

Notas sobre método "post"

  • Este método envia dados do formulário como uma transação HTTP post
  • Os envios de formulários com o método "post" não podem ser marcados
  • O método "post" é mais robusto e seguro do que o "get", e "post" não tem limitações de tamanho

Formtarget

Especifica um nome ou uma palavra-chave que indica onde exibi a resposta recebida após o envio do formulário.

Notas: Este atributo substitui o atributo target do form.

O formtarget funciona com os seguintes tipos de entrada: enviar e imagem.


Formnovalidade

Especifica que um elemento <input> não deve ser validado quando enviado.

Nota: este atributo substitui o atributo novalidade do form..

O formnovalidade funciona somente com o tipo de entrda enviar.


Novalidade

Quando presente, novalidade especifica que todos os dados do formulário não devem ser validados quando enviados. Especifica que nenhum dado do formulário deve ser validado no envio.
